Project Goal

Your team has been hired to build a cloud contact center solution using Amazon Connect.

The new system must support:

Voice support
Chat support
Intelligent routing
Customer records
Case management
Analytics
Conversational AI
Backend integration

Each team must:

Use one shared AWS root account
Each student must log in using:
An individual IAM user

Each IAM user should have:

AdministratorAccess

This allows:

Accountability
Safe collaboration
User-level auditing

1. Create and configure a new Amazon Connect instance for SkyFleet.

Must include:
Admin user setup
Telephony enabled
Basic configuration completed

2. Users and Routing

Create support teams and queues.

Required Queues:
Technical Support
Billing Support
Sales
Required Users:
At least 2 agents per profile
Appropriate routing profiles

3. Voice Contact Flow

Build a main inbound voice flow. The customer should hear a welcome prompt followed by a menu prompt. When pressing different input

Must include:
- Welcome prompt 
- Customer Whisper flow that notifies customers the call may be recorded
- Begin audio recording for later reporting
- accept input after a menu recording plays. Loop menu response if a user selects to replay the menu
- Any time the call will disconnect, play an appropriate whisper to the customer (We don't want a call to suddenly drop and for the customer to be confused thinking they were ignored!)
- All voice prompts should use Amazon Polly
- This flow should capture contact attributes
- create case automatically
- conditional routing for premium vs non-premium
Menu options:
-Technical Support
-Billing
-Sales
-Repeat this message
Queue routing
Must demonstrate:
-Interact blocks
-Set blocks
-Transfer blocks
- Check blocks
- Logic blocks

4. Chat Contact Flow

Build a chat support flow that includes a welcome message, menu choices, and appropriate queue routing.


5. Set up Customer Profiles

Enable and configure customer profiles and create at least 4 different customers with relevant details such as name, company, and service tier

6. Configure Amazon Cases

Include at least 2 custom case fields that make sense for SkyFleet (such as drone model or issue severity) and create a custom case template that includes the custom case fields.

7. Amazon Lex Integration

Create an Amazon Lex bot with at least 2 intents: Technical Issue & Billing Issue. Lex must be invoked from a contact flow.

8. Create at least one lambda function that integrates with the contact flow. Examples may be that we lookup the customer tier or return service status. Get creative here and show off what you can do! One of your flows (either the chat or voice) must demonstrate this function in use.

9. Enable Contact Lens and create a dashboard with details about either the voice or chat flow. Remember that we must have contact recording or transcription enabled in one of our flows to make this work! Create at least one rule to detect phrases such as "Drone failed" or "Urgent Issue". This is another chance to get creative and show how advanced your rules can get!

For the Dashboard, include at least:
Contact volume
Queue metrics
Case metric
^these are flexible and metrics should MAKE SENSE when being grouped on the same dashboard
